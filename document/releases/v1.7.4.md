## Release Note

AndroidBase 1.7.4 adds Flow throttling utilities, a one-time ViewModel initialization helper, and safer CompoundButton two-way binding. It also removes the internal `example` package from the base module to keep the published library focused on public APIs.

### New APIs

- **`FlowExtension.throttleLatest`** — Adds `throttleLatest(periodMillis)` for both `Flow<T>` and `StateFlow<T>`. During an active throttle window, intermediate emissions are dropped and only the latest value is delivered when the window ends. Use this to limit high-frequency updates such as search input, scroll events, or rapid state changes without losing the most recent value.
- **`ViewModelExtension.initializeOnce`** — Adds `initializeOnce { }` on `ViewModel`. The block runs at most once per ViewModel instance on a background dispatcher, making it suitable for startup work such as loading configuration or preparing initial data without repeating on configuration changes.
- **`ViewBindingRefreshAdapter.getItemOrNull`** — Adds `getItemOrNull(position)` to safely read an item from the adapter's current list. Returns `null` when the position is out of bounds, avoiding exceptions in click handlers or item-level logic.

### Behavior Changes

- **`bindCompoundButton` two-way binding** — Reworked the `MutableStateFlow<Boolean>` ↔ `CompoundButton` binding to prevent feedback loops more reliably. User interactions now update the `StateFlow` directly, while UI updates are driven by flow observation. The optional `changeAction` callback is now a `suspend` function and is invoked from the lifecycle-aware collector rather than from the checked-change listener. The listener is also cleared automatically when the owner lifecycle is destroyed, reducing the risk of leaks after the view is torn down.

### Breaking Changes

- **Removed `com.bonepeople.android.base.example` package** — All example classes, adapters, and layouts under `base/src/main/java/com/bonepeople/android/base/example/` have been removed from the base module. These were reference implementations, not part of the library's public API surface. If your project imported or extended classes such as `SimpleViewBindingActivity`, `SimpleViewBindingFragment`, or the `Simple*Adapter` types, copy the needed code into your app or refer to the equivalent patterns in the `sample` module.

### Migration Notes

- **If you referenced base-module example classes** — Replace imports from `com.bonepeople.android.base.example.*` with your own implementations. Reference patterns are available in the `sample` module (for example, `HomeActivity` and related demo screens).
- **If you use `bindCompoundButton` with `changeAction`** — Update the callback to a `suspend` lambda if it performs asynchronous work. No change is required when using the default empty callback.

## Changelog

- Added `throttleLatest(periodMillis)` for `Flow<T>` and `StateFlow<T>` to limit high-frequency emissions while preserving the latest value after each throttle window.
- Added `ViewModel.initializeOnce { }` to run startup work once per ViewModel instance on a background dispatcher.
- Added `ViewBindingRefreshAdapter.getItemOrNull(position)` for bounds-safe item access.
- Reworked `bindCompoundButton` two-way binding with `updatingFromFlow` loop prevention, suspend `changeAction`, and lifecycle destroy cleanup.
- Removed the base `example` package and relocated reference patterns to the sample module.
- Sample: introduced launch flow with `StartActivity`, system splash screen, and startup ViewModel; rebuilt home navigation as `HomeActivity` with a card-based module list and `finishAndRemoveTask` exit behavior.
- Sample: reorganized demo modules under the `test` package, removed Flow and General Test pages, and redesigned List Adapter and UserManager demo screens.
- Sample: refreshed app visual identity with primary color palette, launcher icon, start screen brand animation, and updated home/demo layouts.
